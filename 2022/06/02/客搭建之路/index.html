博客搭建之路

发表于 2022-06-02 更新于 2022-06-05 分类于 2022-06 阅读次数： Valine：
本文字数： 1.9k 阅读时长 ≈ 2 分钟

无论任何时候，使用框架都是一种效率很高的开发方式，特别是在互联网各类技术发展成熟的今天，好的插件、好的框架能让开发者在最短的时间内开发出一款高质量系统。

一、博客系统项目开发的前期准备

1、一个设想

（1）、初衷

尽管本人的编程能力并不强，但大学四年下来也学了不少的技术和知识，无论是生活中的技能还是学习中学到的知识，突然觉得都应该记录一下。 也想着能记录一下我和我的小女朋友王乐宁同学的学习和生活，或是日记、或是笔记，以后的生活中遇到难事、遇到委屈，也能看看过去的文字和心态，或多或少都是安慰~

（2）、工具

大学四年计算机学习感受最深的一点就是——开发系统要找好工具并勤加思考。所以一有这个想法去开发博客系统之后 我做的第一件事就是在网络中寻找开发工具，于是我在知乎中找到一个好的博客系统项目hexo+github pages，同时为了保证我和宁儿对项目的共同管理，初步打算将hexo未来部署到云服务器中，这样就实现了随时随地地编写博客系统。
于是初步选定的工具就是———hexo博客框架、github pages以及腾讯云服务器。

（3）初步计划

跟随前辈的开发步骤对该博客系统进行初步的开发。
通过github将该项目部署到云服务中。
通过访问域名 dns解析实现域名方式访问服务器。
博客中增加自己的功能 实现博客随时随地编写。
开始使用。

2、项目开发

（1）环境搭建

github + git + node.js + hexo

（2）详细步骤
利用SSH完成github的绑定并提交文件（github本身的上传文件很慢）

SSH协议简介：
SSH（安全外壳协议，Secure Shell的缩写）是建立在应用层基础上的安全协议。SSH是目前较为可靠，专为远程登录会话和其他网络服务提供安全性的协议，利用SSH协议可以有效防止远程管理过程中的信息泄露，简单来说，SSH就是保障你的账户安全，将你的数据加密压缩，不仅防止其他人截获你的数据，还能加快传输速度。
详细了解关注该文章详述SSH原理及其应用。
PS：后期需要用到的话可以看这篇文章再做总结
- 利用SSH绑定github：git上传文件到github首先得利用SSH登录远程主机，登录方式有两种，口令登录和公钥登录。口令登录比较麻烦，每次登录都得输入口令，公钥登录则省去的输入密码的步骤。 

- 公钥授权：首先在github上添加SSH key配置，想生成SSH key，就要先安装SSH。在安装Git Bash 自带了SSH，打开Git Bash输入ssh命令，若存在使用命令，则说明已经安装，输入ssh-keygen -t rsa，表示我们指定RSA算法生成密钥，然后敲四次回车，生成两个文件，分别为密钥id_rsa和公钥id_rsa.pub，利用该文件中的key添加到github完成公钥授权。  

- github添加公钥：进入github主页，点击setting，点击SSH and GPG keys，点击New SSH key，将复制的公钥粘贴到key内，点击添加，然后在Git Bash中输入ssh -T git@github.com进行检验，根据英文提示输入yes完成添加。

- 提交文件：提交文件有两种情况，一种是本地没有git仓库，另一种是本地有git仓库，并已经进行了多次commit操作。
    - 本地没有git仓库
      1. 直接将远程仓库clone到本地
      2. 将文件添加并commit到本地仓库
      3. 将本地仓库的内容push到远程仓库
    - 本地有git仓库，并有多次commit
      1. 建立一个本地仓库进入，init初始化
      2. 关联远程仓库
      3. 同步远程仓库和本地仓库
      4. 将文件添加提交到本地仓库
      5. 将本地仓库的内容push到远程仓库
          - 第一种方法步骤：
        进入个人主页，进入自己建立的博客文件项目库，点击clone or download复制地址，进入准备存储的git仓库目录，输入 "git clone 仓库地址" 将远程仓库clone到本地，clone过后可以看到本地仓库和远程仓库内容一致，测试上传文件请求：在本地库中创建一个测试文件text.txt，Git Bash 中输入git status命令查看仓库状态，知当前库中存在一个未被跟踪文件即text.txt（意为没有提交到本地仓库），使用git add 文件名将文件添加到了临时缓冲区，git commit -m "注释"提交到本地仓库
