微服务学习总结

发表于 2022-07-02 更新于 2022-07-03 分类于 微服务学习 阅读次数： Valine：
本文字数： 5.1k 阅读时长 ≈ 5 分钟

本文主要针对微服务架构中的spring cloud框架做基本的使用说明
教程使用的组件如下：
注册中心： nacos 替代方案eureka、consul、zookeeper
配置中心： nacos 替代方案sc config, consul config
服务调用：feign 替代方案 resttempate
熔断：sentinel 替代方案 Resilience4j
负载均衡：sc loadbalance
网关：spring cloud gateway
链路：spring cloud sleuth+zipkin 替代方案：skywalking

教程的总体框架如下：

使用nacos作为注册中心

下载nacos并启动

Nacos致力于帮助您发现、配置和管理微服务。Nacos提供了一组简单易用的特性集，帮助您快速实现服务发现、服务配置、服务元数据及流量管理。
***nacos下载 ***

下载完成解压后，在文件的/bin目录下，windows系统点击startup.cmd启动nacos，高版本的默认nacos的打开方式为cluster，而cluster模式在没有进行cluster配置的情况下，启动会失败。

  • 解决方案：以记事本方式打开startup.cmd，修改其启动模式 set MODE=”standalone” 即单机模式启动nacos，nacos登录页面：http://localhost:8848/nacos/。
